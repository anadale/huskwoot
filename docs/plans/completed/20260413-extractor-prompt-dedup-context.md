# Улучшение промпта экстрактора: дедубликация и уточнение контекста

## Overview
- Две проблемы с качеством извлечения задач из текстов
- **Дедубликация**: в meeting summaries одно обещание упоминается несколько раз (в ходе обсуждения и в итогах), модель извлекает его дважды из одного вызова Extract
- **Избыточный контекст**: модель генерирует context, который просто перефразирует summary, не добавляя новой информации
- Обе проблемы решаются правкой системного промпта экстрактора, без изменений в коде

## Context (from discovery)
- Файл для правки: `internal/ai/extractor.go`, константа `extractorSystemTmpl` (строки 52-83)
- Тесты: `internal/ai/extractor_test.go` — используют мок-ответы, не зависят от текста промпта
- Промпт уже содержит инструкцию «если контекст очевиден из summary — оставь пустой строкой», но модель не следует ей из-за нечёткой формулировки

## Development Approach
- **testing approach**: Regular — изменения только в тексте промпта, кодовая логика не меняется
- Существующие тесты покрывают парсинг/фильтрацию результатов, а не содержимое промпта
- После правки: прогнать `go test ./...` и `go vet ./...` для подтверждения, что ничего не сломано

## Testing Strategy
- **unit tests**: существующие тесты в `internal/ai/extractor_test.go` должны пройти без изменений
- Новые тесты не требуются — изменение влияет на поведение LLM, а не на кодовую логику

## Technical Details

### Блок дедубликации (добавить после «НЕ извлекай»)
```
Дедубликация:
- Если одно и то же обещание упоминается в тексте несколько раз (например,
  в ходе обсуждения и в итогах/action items), извлеки его только один раз
- Предпочитай формулировку из итогов или финального упоминания — она обычно
  точнее и полнее
```

### Замена блока контекста
**Было:**
```
Правила формирования context:
- Краткая справка (1–2 предложения), позволяющая понять суть обещания без исходного сообщения
- Кто попросил, по какому поводу, в чём проблема — только факты из текста
- Если контекст очевиден из summary или отсутствует — оставь пустой строкой
```

**Стало:**
```
Правила формирования context:
- Краткая справка (1–2 предложения), позволяющая понять суть обещания без исходного сообщения
- Кто попросил, по какому поводу, в чём проблема — только факты из текста
- Контекст должен добавлять информацию, которой НЕТ в summary
- Если context лишь перефразирует summary (кто именно собирается сделать то,
  что уже описано в summary, или повторяет сроки) — оставь пустой строкой
- Если в тексте нет дополнительных фактов (кто попросил, почему, предыстория) — оставь пустой строкой
```
