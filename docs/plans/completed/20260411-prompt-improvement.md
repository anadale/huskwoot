# Переработка промптов детектора и экстрактора обещаний

## Overview

Текущие промпты детектора и экстрактора слишком просты — не содержат определений, правил и ограничений. Это приводит к false positives (описания фич воспринимаются как обещания) и false negatives (реальные обещания пропускаются). Кроме того, Extractor возвращает одну задачу, тогда как в email (meeting summaries) может быть 0..N обещаний.

Изменения:
- Расширенные промпты с таксономией обещаний и контр-примерами
- Extractor возвращает массив задач вместо одной
- Pipeline итерирует по массиву

## Technical Details

### Новые промпты

**Детектор (системный):**
```
Ты анализируешь текст на наличие обещаний пользователя {{.UserName}}{{if .Aliases}} (также известен как: {{.Aliases}}){{end}}.

Обещание — это когда {{.UserName}} лично берёт на себя обязательство выполнить
конкретное действие. Сюда входят:
- явные обещания: «пообещал», «согласился сделать», «взялся», «обязался»
- планы с конкретным сроком: «планирует сделать к пятнице», «к концу недели запустит»

НЕ являются обещаниями:
- описания функциональности, фич или продуктов
- предложения и рекомендации другим людям («предложил Алексею...»)
- просьбы и поручения другим («попросил Викторию прислать...»)
- факты о прошлом («разработал скрипт», «Алексей работает над задачами»)
- планы без конкретного срока («планирует обучить нейросети»)
- действия и обязательства других людей

Ответь только одним словом: yes или no
```

**Детектор (пользовательский):**
```
{{if .ReplyTo}}Сообщение, на которое отвечает пользователь:
"{{.ReplyTo.Text}}" (от {{.ReplyTo.AuthorName}})

{{end}}Текст для анализа:
{{if .Reaction}}Реакция {{.Reaction.Emoji}}{{else}}{{.Text}}{{end}}
```

**Экстрактор (системный):**
```
Извлеки обещания пользователя {{.UserName}}{{if .Aliases}} (также известен как: {{.Aliases}}){{end}} из текста.

Обещание — это когда {{.UserName}} лично берёт на себя обязательство выполнить
конкретное действие:
- явные обещания: «пообещал», «согласился сделать», «взялся», «обязался»
- планы с конкретным сроком: «планирует сделать к пятнице», «к концу недели запустит»

НЕ извлекай:
- описания функциональности, фич или продуктов
- предложения и рекомендации другим людям
- просьбы и поручения другим
- факты о прошлом (уже выполненные действия)
- планы без конкретного срока
- действия и обязательства других людей

Правила формирования summary:
- Опиши конкретное действие от лица пользователя: «загрузить мартовские данные скриптом»
- Не включай имя пользователя в summary
- Не повторяй контекст, только само действие и его цель

Верни JSON-массив без дополнительного текста. Если обещаний нет — верни [].
[{"summary": "что нужно сделать", "deadline": "ISO 8601 или null", "confidence": 0.0-1.0}]
```

**Экстрактор (пользовательский):**
```
{{if .History}}История диалога:
{{range .History}}[{{.Timestamp.Format "2006-01-02 15:04"}}] {{.AuthorName}}: {{.Text}}
{{end}}
{{end}}Текст для анализа:
{{if .Reaction}}Реакция {{.Reaction.Emoji}}{{if .ReplyTo}} на "{{.ReplyTo.Text}}"{{end}}{{else}}{{.Text}}{{end}}

Текущая дата и время: {{.Now}}
```

### Изменение сигнатуры

```go
// Было:
Extract(ctx context.Context, msg Message, history []Message) (*Task, error)

// Стало:
Extract(ctx context.Context, msg Message, history []Message) ([]Task, error)
```

### Генерация ID задач

```go
task := model.Task{
    ID: msg.Source.ID + "/" + msg.ID + "/" + strconv.Itoa(i),
    // ...
}
```

Индекс `i` обеспечивает уникальность при нескольких задачах из одного сообщения/письма.
